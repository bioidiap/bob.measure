bob.measure.brute_force.eer_threshold(negatives, positives, is_sorted=False)[source]¶

Calculates threshold as close as possible to the equal error rate (EER)

The EER should be the point where the FPR equals the FNR. Graphically, this would be equivalent to the intersection between the ROC (or DET) curves and the identity.

Note

The scores will be sorted internally, requiring the scores to be copied. To avoid this copy, you can sort both sets of scores externally in ascendant order, and set the is_sorted parameter to True.

Parameters

  • negatives (numpy.ndarray (1D, float)) – The set of negative scores to compute the threshold

  • positives (numpy.ndarray (1D, float)) – The set of positive scores to compute the threshold

  • is_sorted (bool, Optional) – Set this to True if the negatives are already sorted in ascending order. If False, scores will be sorted internally, which will require more memory.

Returns

threshold – The threshold (i.e., as used in fprfnr()) where FPR and FNR are as close as possible

Return type

float

bob.measure.brute_force.min_weighted_error_rate_threshold(negatives, positives, cost, is_sorted=False)[source]¶

Calculates the threshold that minimizes the error rate

The cost parameter determines the relative importance between false-accepts and false-rejections. This number should be between 0 and 1 and will be clipped to those extremes. The value to minimize becomes: \(ER_{cost} = cost * FPR + (1-cost) * FNR\). The higher the cost, the higher the importance given to not making mistakes classifying negatives/noise/impostors.

Note

The scores will be sorted internally, requiring the scores to be copied. To avoid this copy, you can sort both sets of scores externally in ascendant order, and set the is_sorted parameter to True.

Parameters

  • negatives (numpy.ndarray (1D, float)) – The set of negative and positive scores to compute the threshold

  • positives (numpy.ndarray (1D, float)) – The set of negative and positive scores to compute the threshold

  • cost (float) – The relative cost over FPR with respect to FNR in the threshold calculation

  • is_sorted (bool, Optional) – Set this to True if the positives are already sorted in ascending order. If False, scores will be sorted internally, which will require more memory.

Returns

threshold – The threshold for which the weighted error rate is minimal

Return type

float
